You're right — the backtest uses a FIXED BASE_MARGIN_USD = 6.4 throughout all
27.7 months. It doesn't compound. The account grows on paper but new grids
always open at $6.4 L1, not a recalculated fraction of the current balance.

So the real question is: what's the right compounding rule? Two options:

1. Fixed %: BASE_MARGIN = X% of current account balance, recalculated each new
   grid open. Fully compounds — position size grows as you win, shrinks if you lose.

2. Fixed floor + periodic manual bumps: Keep $6.4 until account hits some
   threshold (e.g. $600), then bump to $9.6, etc. Simpler, more controlled.

Option 1 is cleaner to backtest. Want me to run it with e.g. BASE_MARGIN =
1.6% of account (which starts at $6.4 on a $400 account) and see what the
compounded 30x result actually looks like?
